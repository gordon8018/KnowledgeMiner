- 选项1: 修改函数签名接受第二个参数
- 选项2: 修改调用代码只传递concept

## ✅ HIGH — 架构问题 (4/4 确认)

### 6. ✅ 双 Config 系统
**位置**: `src/config.py` vs `src/core/config.py`
**问题**: 两个完全独立的Config系统，Phase 1和Phase 2+使用不同配置
**影响**: 配置混乱、不一致、难以维护
**验证结果**: **CONFIRMED**

```python
# src/config.py (Phase 1) - dataclass
@dataclass
class Config:
    model_name: str = "gpt-3.5-turbo"
    # 旧式配置

# src/core/config.py (Phase 2+) - Pydantic
class LLMConfig(BaseSettings):
    model: str = "claude-sonnet-4-6"  # 不同的默认模型！
    # 新式配置
```

**修复建议**: 统一配置系统，迁移到Pydantic配置

### 7. ✅ ConceptType 重复定义
**位置**: `src/models/concept.py:5` vs `src/core/concept_model.py:12`
**问题**: 语义相同但类型不兼容
**影响**: 类型检查失败、代码混乱
**验证结果**: **CONFIRMED**

```python
# src/models/concept.py
class ConceptType(Enum):
    TERM = "term"
    INDICATOR = "indicator"
    # ...

# src/core/concept_model.py
class ConceptType(str, Enum):  # 继承str！
    TERM = "term"
    INDICATOR = "indicator"
    # ...
```

**修复建议**: 保留一个定义，推荐使用 `ConceptType(str, Enum)`

### 8. ✅ Concept 被错误传入 CandidateConcept 参数
**位置**: `src/main.py:206-207`
**问题**: 内层循环把Concept对象当CandidateConcept传入
**影响**: 类型不匹配、潜在运行时错误
**验证结果**: **CONFIRMED**

```python
# 问题代码

    for candidate in concepts:  # 也是Concept对象！
        self.relation_mapper.add_candidate_concept(candidate)  # 期望CandidateConcept

@dataclass

    name: str
    type: ConceptType
    confidence: float
    # ...

    name: str
    type: ConceptType
    definition: str
    # ...
```

**修复建议**: 修正循环逻辑或修改参数类型

**验证结果**: **CONFIRMED**

- `pattern_detector.py`: 实现了3种模式
- `gap_analyzer.py`: 实现了3种模式
- `insight_generator.py`: 实现了3种模式

**修复建议**: 提取共同逻辑到基类或工具函数
